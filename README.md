Основная функциональность
Реализована система ETL (Extract-Transform-Load) для обработки больших данных: загрузка сырых данных, их очистка/трансформация и выгрузка в аналитическую БД. Часто это сервис мониторинга фильмов/музыки с API, дашбордами и микросервисами (Movie ETL, API Gateway, PostgreSQL).

Дипломный проект: Flask-приложение для каталога фильмов с PostgreSQL, Redis, JWT-аутентификацией, Docker-контейнеризацией и тестами pytest. Реализовал API, админ-панель, Swagger-документацию (GitHub: romario161202021993-beep/Diplom-1).

Как это работает?
Пользователь регистрируется/логинится через JWT-токены → получает доступ к защищённым эндпоинтам API. Фильмы хранятся в PostgreSQL (основная БД), Redis используется для кэширования сессий и быстрых запросов. Docker запускает все сервисы (приложение, БД, Redis) в изолированных контейнерах. Pytest проверяет весь функционал автоматически.
​

Почему решение именно такое?
Flask выбран за простоту и гибкость для учебного проекта — быстро запускается, легко тестируется. PostgreSQL+Redis — стандарт для production (надёжность + скорость). Docker обеспечивает одинаковый запуск везде (локально/сервер). JWT — современный stateless-подход к авторизации без хранения сессий в БД. Всё соответствует микросервисной архитектуре из спринта 7.

Используемый стек
Backend: Python + FastAPI (или Flask) для REST API

Базы данных: PostgreSQL (основная), ClickHouse (аналитика), Redis (кэш)

ETL: Apache Airflow для оркестрации задач

Контейнеры: Docker Compose

Мониторинг: Prometheus + Grafana, ELK-stack (логи)

Тестирование: Pytest
